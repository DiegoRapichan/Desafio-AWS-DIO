# Desafio-AWS-DIO
Desafio Proposto no Bootcamp da DIO, na qual o objetivo é oferecer e explicar 3 tecnologias AWS para uma empresa de distribuição farmacêutica que não utiliza serviço de cloud

RELATÓRIO DE IMPLEMENTAÇÃO DE SERVIÇOS AWS
Data: 20/01/2026
Empresa: Abstergo Industries
Responsável: Diego Colombari Rapichan
Introdução
Este relatório apresenta o processo de implementação de serviços na empresa Abstergo Industries, realizado por Diego Colombari Rapichan. O objetivo do projeto foi elencar três serviços AWS, com foco na redução de custos imediatos e aumento de eficiência operacional.
Descrição do Projeto
O projeto de implementação de ferramentas foi dividido em 3 etapas, cada uma com seus objetivos específicos. A seguir, são descritas as etapas do projeto:
Etapa 1
    • Nome da ferramenta: Amazon EC2 com Auto Scaling
    • Foco da ferramenta: Gerenciamento de servidores virtuais escaláveis sob demanda.
    • Descrição de caso de uso: A Abstergo precisa hospedar sistemas internos, como ERP e comunicação com parceiros, que têm variação de demanda ao longo do mês. Com o Auto Scaling, os servidores aumentam ou diminuem automaticamente conforme a demanda, garantindo que a empresa pague apenas pelo que realmente utiliza, evitando gastos com servidores ociosos.
Etapa 2
    • Nome da ferramenta: Amazon RDS (Relational Database Service)
    • Foco da ferramenta: Banco de dados relacional gerenciado, seguro e escalável.
    • Descrição de caso de uso: A empresa mantém dados de estoque, pedidos e parceiros. Com RDS, não é necessário manter um DBA dedicado para manutenção de banco, backups ou atualizações de software, o que reduz custos operacionais. Além disso, o RDS oferece pagamento por uso e instâncias reservadas, permitindo otimizar gastos conforme o crescimento da empresa.
Etapa 3
    • Nome da ferramenta: Amazon S3 (Simple Storage Service)
    • Foco da ferramenta: Armazenamento de arquivos seguro, escalável e de baixo custo.
    • Descrição de caso de uso: A Abstergo lida com grande volume de documentos fiscais, notas de pedidos e arquivos de logística. Com S3, é possível armazenar todos esses dados de forma econômica e acessar apenas quando necessário, sem a necessidade de manter servidores de armazenamento próprios. A integração com políticas de ciclo de vida permite mover dados antigos para camadas de armazenamento mais baratas, reduzindo ainda mais os custos.
Conclusão
A implementação de serviços na empresa Abstergo Industries trouxe benefícios significativos, como:
    • Redução de custos com servidores ociosos via Auto Scaling.
    • Menor gasto com manutenção de bancos de dados com RDS.
    • Armazenamento eficiente e barato de documentos via S3.
Essas ferramentas aumentam a eficiência e a produtividade da empresa. Recomenda-se a continuidade da utilização das ferramentas implementadas e a busca por novas tecnologias que possam melhorar ainda mais os processos da empresa.
